# Дашборд статистики Башкортостана

Веб-проект для просмотра официальной статистики по районам и городским округам Республики Башкортостан. В первой версии есть дашборд, рейтинг районов по выбранному показателю, карточка района, справочник источников, журнал импорта и проверки качества данных.

## Что уже реализовано

- Справочник всех 63 муниципалитетов верхнего уровня Республики Башкортостан: 54 муниципальных района и 9 городских округов.
- Расчет места района в рейтинге по выбранному году и показателю.
- Поддержка направлений рейтинга: больше лучше, меньше лучше, нейтрально.
- API для районов, показателей, значений, рейтингов, источников, качества данных и импортов.
- React-дэшборд с фильтрами по году, району, группе и показателю.
- Единый дашборд с демо-входом по роли: руководство региона или руководитель выбранного района.
- Руководительская сводка: ключевые показатели, худшие позиции и проблемные сигналы по районам.
- Официальный статический снимок БД ПМО Росстата через обработку `tochno.st`: 21 раздел, 178 показателей и 14 731 значение по районам и городским округам.
- Оперативный слой 2026 из портала открытых данных Республики Башкортостан: 14 показателей по объектам здравоохранения, опубликовано 28.01.2026.
- Импорт CSV из официальных источников через настраиваемые URL.
- Импорт плоской выгрузки БД ПМО Росстата по муниципальным образованиям.
- Автоматическое пополнение справочника групп и показателей из БД ПМО, чтобы загрузить полный каталог из 603 показателей.
- Импорт точных показателей Федстат/ЕМИСС через SDMX и явные ID фильтров.
- Проверки данных: неизвестный район, неизвестный показатель, некорректный год, некорректное значение, дубли и пропуски.

GitHub Pages публикует статическую демо-версию дашборда:

`https://konstantinoviceduard9-bit.github.io/bashkortostan-stats-dashboard/`

На GitHub Pages Express API не запускается, поэтому опубликованная версия использует встроенный официальный снимок БД ПМО. Демонстрационные статистические значения в пользовательском дашборде не используются.

Текущий снимок БД ПМО включает годы 2023-2025. Для 2026 года добавлен отдельный оперативный слой по открытым данным Республики Башкортостан. Это не замена БД ПМО: у каждого 2026-показателя указан отдельный источник и дата публикации.

Разделы 31 “Население” и 32 “Занятость и заработная плата” у `tochno.st` поставляются отдельными большими архивами 3.7 ГБ и 1.3 ГБ, поэтому они подключаются отдельным фоновым этапом.

## Структура проекта

- `src/client/` - интерфейс дашборда.
- `src/server/` - Express API, импорт данных и хранилище.
- `src/shared/` - общие типы, расчет рейтингов и проверки качества.
- `src/data/seed.ts` - стартовые справочники и демонстрационные значения.
- `src/data/schema.sql` - SQL-схема будущей базы данных.
- `src/data/example-official.csv` - пример CSV для официального источника.
- `src/data/example-bdmo-flat.csv` - пример плоской CSV-выгрузки БД ПМО.
- `src/data/example-fedstat-spec.json` - пример точной спецификации показателя Федстата.
- `tests/` - тесты рейтинга, качества данных и импорта.

## Основные API

- `GET /api/districts` - районы и городские округа.
- `GET /api/indicator-groups` - группы показателей.
- `GET /api/indicators` - показатели, можно фильтровать `groupId`.
- `GET /api/years` - доступные годы.
- `GET /api/rankings?indicatorId=average_salary&year=2024` - рейтинг районов.
- `GET /api/districts/ufa/profile?year=2024` - карточка района.
- `GET /api/executive-summary?year=2024` - сводка для руководства региона.
- `GET /api/sources` - официальные источники.
- `GET /api/quality` - отчет о качестве данных.
- `GET /api/import-runs` - журнал загрузок.
- `GET /api/import-sources` - настроенные источники для автоматического импорта.
- `POST /api/imports/official/:sourceId` - запуск импорта из URL, заданного в `.env`.
- `POST /api/imports/csv-url` - разовый импорт из переданного URL.
- `POST /api/imports/fedstat` - импорт точного показателя Федстат/ЕМИСС по SDMX-спецификации.

## Единый дашборд и роли

Отдельный дашборд руководителя не создаем. Вместо этого используем один продукт с разными представлениями:

- `Руководство региона` - видит общую картину по всем районам, ключевые показатели и проблемные сигналы.
- `Муниципалитет` - выбирает свой район на входе и видит фокус на своем районе: значения, место в рейтинге и сравнение с другими.
- `Администратор` - в следующем этапе получит управление импортами, источниками и справочниками.

Сейчас вход демонстрационный: кнопка “Войти как руководство региона” задает роль пользователя в интерфейсе. Позже этот слой заменяется на настоящую авторизацию с логинами, паролями, ролями и привязкой пользователя к району.

Для районных руководителей на стартовом экране есть отдельный блок “Вход руководителя района”: пользователь выбирает район, после чего фильтр района фиксируется, а дашборд показывает текущее место района по выбранному показателю.

Руководительский блок внутри общего дашборда показывает:

- ключевые показатели за выбранный год;
- районы в нижней части рейтингов;
- критические сигналы, если район оказался внизу рейтинга;
- переход по клику сразу к нужному району и показателю.

### Простой формат проекта

CSV должен содержать заголовки:

```csv
district,indicator,year,value,source
Уфа,population_total,2024,1144000,rosstat
Белорецкий район,average_salary,2024,56000,rosstat
```

Поля:

- `district` - название района или городского округа, как в справочнике.
- `indicator` - технический код показателя.
- `year` - год значения.
- `value` - числовое значение.
- `source` - код источника, например `rosstat`, `bashstat`, `budget_open_data`.

### Формат БД ПМО Росстата

Основной официальный источник для муниципальной статистики - БД ПМО Росстата:

`https://rosstat.gov.ru/free_doc/new_site/bd_munst/munst.htm`

Для первой реальной загрузки поддерживается плоская CSV-таблица с разделителем `;`. Минимальные поля: `region`, `mo`, `indicator`, `year`, `value`. Поддерживается и формат `tochno.st` на базе БД ПМО Росстата: `region_name`, `municipality`, `indicator_section`, `indicator_section_code`, `indicator_code`, `indicator_name`, `year`, `indicator_value`, `indicator_unit`.

Для полного каталога 603 показателей желательно также передавать `section`, `indicator_code` и `unit`:

```csv
region;section;mo;oktmo;indicator_code;indicator;unit;year;value
Республика Башкортостан;31. Население;Уфа;80701000001;31010000;Численность населения;чел.;2024;1144000
Республика Башкортостан;32. Занятость и заработная плата;Белорецкий район;80611000000;32010000;Средняя зарплата;руб.;2024;56000
```

Парсер берет только строки, где `region`/`region_name` равен `Республика Башкортостан`, сопоставляет название муниципалитета со справочником районов и создает новые показатели автоматически, если их еще нет в локальном справочнике.

### Стратегия полного каталога БД ПМО

В БД ПМО доступно 603 муниципальных показателя в 33 разделах. Для максимального охвата загружаем не отдельные ручные показатели, а полную плоскую выгрузку:

1. Берем CSV/XLSX выгрузку БД ПМО, преобразованную в плоский формат.
2. Оставляем строки только по `Республика Башкортостан`.
3. Для каждого нового `section` создаем группу показателей.
4. Для каждого нового `indicator_code` создаем показатель с единицей измерения.
5. Все значения сохраняем по району, году, показателю и источнику.
6. После импорта дашборд сразу может строить рейтинг по любому загруженному показателю.

Если в выгрузке нет `indicator_code`, система создаст ID из названия показателя. Для точности лучше использовать `indicator_code`, потому что названия могут немного меняться между версиями.

## Настройка официальных источников

Скопируйте `.env.example` в `.env` и укажите прямые ссылки на CSV:

```env
ROSSTAT_CSV_URL=https://example.gov/stat.csv
BASHSTAT_CSV_URL=https://example.gov/bashstat.csv
BUDGET_OPEN_DATA_CSV_URL=https://example.gov/budget.csv
BDMO_FLAT_CSV_URL=https://example.gov/bdmo-flat.csv
```

Затем запустите API и вызовите:

```bash
curl -X POST http://127.0.0.1:4100/api/imports/official/rosstat
```

Для плоской выгрузки БД ПМО:

```bash
curl -X POST http://127.0.0.1:4100/api/imports/official/bdmo_rosstat
```

Если формат источника изменится, нужно обновить только адаптер импорта в `src/server/importer.ts`, а не весь дашборд.

## Точные показатели Федстат / ЕМИСС

Федстат не стоит подключать по свободному текстовому поиску: названия похожи, фильтры зависят от страницы показателя, а ID значений хранятся во внутренней структуре Fedstat. Поэтому импорт работает только по явной спецификации.

Минимальный пример:

```json
{
  "format": "sdmx",
  "sourceId": "fedstat",
  "fedstatIndicatorId": "37426",
  "title": "Название показателя в Федстате",
  "localIndicatorId": "population_total",
  "municipalityDimension": "municipality",
  "yearDimension": "year",
  "selectedFilterIds": ["0_37426", "1_2024", "2_Республика Башкортостан"],
  "filterObjectIds": {
    "lineObjectIds": "1",
    "columnObjectIds": "2",
    "filterObjectIds": "0"
  },
  "districtAliases": {
    "Уфа": "ufa",
    "Белорецкий район": "beloretsk"
  }
}
```

Отправка:

```bash
curl -X POST http://127.0.0.1:4100/api/imports/fedstat \
  -H "Content-Type: application/json" \
  --data @src/data/example-fedstat-spec.json
```

Как получать точные ID:

1. Открыть страницу показателя на `https://www.fedstat.ru/indicator/<id>`.
2. Зафиксировать `fedstatIndicatorId`, название показателя и нужные фильтры.
3. Получить `selectedFilterIds` и `filterObjectIds` из страницы показателя или через инструмент, совместимый с `fedstat_get_data_ids`.
4. Сохранить спецификацию вместе с проектом, чтобы было видно, какой именно показатель и фильтры использовались.

Такой подход медленнее на настройке, зато исключает подмену похожих показателей и сохраняет воспроизводимость загрузки.
