# Дашборд статистики Башкортостана

Веб-проект для просмотра статистики по районам и городским округам Республики Башкортостан. В первой версии есть дашборд, рейтинг районов по выбранному показателю, карточка района, справочник источников, журнал импорта и проверки качества данных.

## Что уже реализовано

- Справочники районов, групп показателей, показателей и источников.
- Расчет места района в рейтинге по выбранному году и показателю.
- Поддержка направлений рейтинга: больше лучше, меньше лучше, нейтрально.
- API для районов, показателей, значений, рейтингов, источников, качества данных и импортов.
- React-дэшборд с фильтрами по году, району, группе и показателю.
- Импорт CSV из официальных источников через настраиваемые URL.
- Импорт плоской выгрузки БД ПМО Росстата по муниципальным образованиям.
- Проверки данных: неизвестный район, неизвестный показатель, некорректный год, некорректное значение, дубли и пропуски.

## Структура проекта

- `src/client/` - интерфейс дашборда.
- `src/server/` - Express API, импорт данных и хранилище.
- `src/shared/` - общие типы, расчет рейтингов и проверки качества.
- `src/data/seed.ts` - стартовые справочники и демонстрационные значения.
- `src/data/schema.sql` - SQL-схема будущей базы данных.
- `src/data/example-official.csv` - пример CSV для официального источника.
- `src/data/example-bdmo-flat.csv` - пример плоской CSV-выгрузки БД ПМО.
- `tests/` - тесты рейтинга, качества данных и импорта.

## Основные API

- `GET /api/districts` - районы и городские округа.
- `GET /api/indicator-groups` - группы показателей.
- `GET /api/indicators` - показатели, можно фильтровать `groupId`.
- `GET /api/years` - доступные годы.
- `GET /api/rankings?indicatorId=average_salary&year=2024` - рейтинг районов.
- `GET /api/districts/ufa/profile?year=2024` - карточка района.
- `GET /api/sources` - официальные источники.
- `GET /api/quality` - отчет о качестве данных.
- `GET /api/import-runs` - журнал загрузок.
- `GET /api/import-sources` - настроенные источники для автоматического импорта.
- `POST /api/imports/official/:sourceId` - запуск импорта из URL, заданного в `.env`.
- `POST /api/imports/csv-url` - разовый импорт из переданного URL.

## Формат CSV для импорта

### Простой формат проекта

CSV должен содержать заголовки:

```csv
district,indicator,year,value,source
Уфа,population_total,2024,1144000,rosstat
Белорецкий район,average_salary,2024,56000,rosstat
```

Поля:

- `district` - название района или городского округа, как в справочнике.
- `indicator` - технический код показателя.
- `year` - год значения.
- `value` - числовое значение.
- `source` - код источника, например `rosstat`, `bashstat`, `budget_open_data`.

### Формат БД ПМО Росстата

Основной официальный источник для муниципальной статистики - БД ПМО Росстата:

`https://rosstat.gov.ru/free_doc/new_site/bd_munst/munst.htm`

Для первой реальной загрузки поддерживается плоская CSV-таблица с разделителем `;`:

```csv
region;mo;oktmo;indicator;year;value
Республика Башкортостан;Уфа;80701000001;Численность населения;2024;1144000
Республика Башкортостан;Белорецкий район;80611000000;Средняя зарплата;2024;56000
```

Парсер берет только строки, где `region` равен `Республика Башкортостан`, сопоставляет название муниципалитета со справочником районов и название показателя со справочником показателей.

## Настройка официальных источников

Скопируйте `.env.example` в `.env` и укажите прямые ссылки на CSV:

```env
ROSSTAT_CSV_URL=https://example.gov/stat.csv
BASHSTAT_CSV_URL=https://example.gov/bashstat.csv
BUDGET_OPEN_DATA_CSV_URL=https://example.gov/budget.csv
BDMO_FLAT_CSV_URL=https://example.gov/bdmo-flat.csv
```

Затем запустите API и вызовите:

```bash
curl -X POST http://127.0.0.1:4100/api/imports/official/rosstat
```

Для плоской выгрузки БД ПМО:

```bash
curl -X POST http://127.0.0.1:4100/api/imports/official/bdmo_rosstat
```

Если формат источника изменится, нужно обновить только адаптер импорта в `src/server/importer.ts`, а не весь дашборд.
