# Полная загрузка БД ПМО для Башкортостана

## Решение

Сначала делаем официальную загрузку БД ПМО Росстата. Демо-значения в статистике не используются. Цель - получить все муниципалитеты верхнего уровня Республики Башкортостан и максимум муниципальных показателей из плоской выгрузки БД ПМО.

## Покрытие

- Муниципалитеты: 54 муниципальных района и 9 городских округов Башкортостана.
- Показатели: полный доступный каталог БД ПМО, до 603 показателей в 33 разделах, если они есть в выгрузке.
- Годы: все годы, которые присутствуют в загружаемой выгрузке. В интерфейсе по умолчанию выбирается самый свежий доступный год.
- Источник: БД ПМО Росстата или плоская обработанная выгрузка на основе БД ПМО с сохранением ссылки и кода показателя.

## Правило качества данных

В статистических карточках, рейтингах и районном режиме нельзя показывать расчетные или демонстрационные значения. Если официальное значение не загружено, интерфейс показывает пустое состояние или сообщение `Нет данных`. Это лучше, чем рейтинг на ненастоящих числах.

## Данные

Система должна принимать плоскую таблицу с полями:

- `region` - субъект РФ, берем только `Республика Башкортостан`.
- `mo` - муниципальное образование.
- `oktmo` - код ОКТМО, если есть.
- `section` - раздел БД ПМО.
- `indicator_code` - код показателя.
- `indicator` - название показателя.
- `unit` - единица измерения.
- `year` - год.
- `value` - числовое значение.

Если часть полей отсутствует, импорт должен продолжать работу для пригодных строк и показывать предупреждения.

## Изменения в проекте

- Расширить справочник муниципалитетов до всех 63 верхнеуровневых муниципалитетов Башкортостана.
- Добавить устойчивые алиасы названий: полные названия БД ПМО, короткие названия, варианты `город ...`, `... муниципальный район`.
- Улучшить импорт БД ПМО так, чтобы новые муниципалитеты и показатели корректно попадали в дашборд.
- Обновить статические данные GitHub Pages так, чтобы опубликованная версия показывала только официальный снимок БД ПМО.
- Удалить или изолировать старые демо-значения, чтобы они не попадали в пользовательский дашборд.
- В README явно указать, что статистика строится только на официально загруженных данных.

## Проверка

- Тест импорта должен подтверждать, что строки по Башкортостану загружаются для нескольких новых районов и городских округов.
- Тест справочника должен подтверждать наличие 63 муниципалитетов.
- Тест рейтинга должен подтверждать, что районный руководитель видит место своего района среди полного списка.
- Перед публикацией выполняются `npm test` и `npm run build`.

## Риски

- На GitHub Pages нет Express API, поэтому для публичной ссылки нужен статический снимок данных.
- Если выгрузка БД ПМО меняет названия столбцов или муниципалитетов, часть строк уйдет в предупреждения до обновления алиасов.
- Реальные данные нельзя заменять расчетными числами, иначе рейтинг будет вводить в заблуждение.
